Development Plan

1. Names

  • Team: Abdullah Khamis, Chaemin Lee, Bavishan Mathan, Jackson Bundy
  • Client: Seojin Ko
  • App: Task Manager

2. Purpose

Our app is going to be a calendar/time management app. The target users are students/employees looking to organize their schedule. They are likely to be mobile users, so the site should be designed with narrower widths first and foremost, although desktop users do need some consideration. The app should allow users to add as many tasks/events as they want and be able to see when/where they are set to happen. They should also be able to view deadlines for projects, and be able to track their progress.

3. Data Description

Task Table

  • Id: Primary key
  • Title: varchar(255) not null
  • Description: text
  • Due_date: timestamp
  • Status: varchar(50) default 'pending' (either 'pending' or 'completed')
  • Created_At: timestamp

Schedule Table

  • Id: Primary key
  • TaskId: references task id
  • Start time: timestamp not null
  • End time: timestamp not null
  • Created_at: timestamp

Progress/Achievements Table

  • Id: Primary key
  • Name: varchar(50) not null
  • Description: varchar(250)
  • Condition_type: varchar(50) not null - indicates condition type such as tasks completed or daily tasks done
  • Condition_value: int not null

4. Functions Performed by App

  • Store tasks in a calendar (INSERT operations)
  • Update task manager as tasks are completed (UPDATE operations)
  • Update achievement page upon reaching milestones (e.g., 10 tasks completed, 20, etc.) (UPDATE operations)
  • Delete task when finished (DELETE operations)

5. Roles of PHP and JS

  • Backend: Handles communication with the database by storing, deleting, or updating user data. It also sends data to the front end when updates are required.
  • Frontend: Focuses on creating a professional user interface and managing the exchange of information with the backend to process or update tasks.

6. Roles of Team Members

  • Abdullah: CSS styling for a polished user interface and writing SQL queries for backend-database communication.
  • Bavishan: Creating HTML templates and assisting with the overall styling of the app.
  • Chaemin: Handling data collection from users and managing its storage or manipulation (PHP, SQL).
  • Jackson: Developing frontend solutions to display data received from the backend.

7. Increments

  • Initial Version: Calendar with addable events
  • Timer and Alerts
  • Task Manager View
  • Contact Book
  • Achievements

8. Client Feedback

“Since students and office workers usually organize their schedules on their phones, I'm satisfied with making an app in a mobile size. I also like organizing my schedule in a calendar format, and it's convenient to manage and update my to-do lists within the calendar. But can you add a timer feature as well? I want to set deadlines for my tasks each day so I can visually see how much time I have left to complete my work.”

Client wants us to add a timer in our app in order to manage the users work efficiently.
